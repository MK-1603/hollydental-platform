========================================================================
TECHNICAL IMPLEMENTATION PLAN
========================================================================

Below is the definitive technical implementation plan generated by the Antigravity engine before codebase execution begins.

### Phase 1: Database & Secure Server Core (The Fortress)
We must establish an enterprise-grade backend before rendering a single UI component.
- Define the Drizzle schema: `users` (patients & staff), `appointments`, `teeth_records`, `invoices`, and `audit_logs`.
- Setup the Express server with `cookie-parser` and `helmet` for Content Security Policies.
- Implement the rigorous Express-Rate-Limit middleware (`globalLimiter`, `authLimiter`, `aiLimiter`).
- Build the JWT issuing and validation logic (`httpOnly`, `SameSite=Lax`).

### Phase 2: Dual-Client Next.js Foundation
The frontend must securely isolate Patient views from Admin views.
- Build the Tailwind CSS theme provider utilizing the clinical light-mode color palette.
- Implement Zustand state management (`authStore`) to handle the dual-role logic.
- Create the overarching Layout components to protect `/patient/*` and `/admin/*` routes.

### Phase 3: The SVG Interactive Dental Chart
A highly complex, mathematically drawn graphical interface.
- Build the `DentalChartSVG` reusable component mapping all 32 adult teeth.
- Implement the click handlers and Zustand state to inject status colors (green/blue/silver).
- Wire the backend `GET /api/patients/:id/chart` endpoint to populate the notes.

### Phase 4: Cloudinary Integration & Multer Intercepts
- Build the `files.routes.js` endpoints.
- Configure Multer to strictly use `memoryStorage()` to bypass the local hard drive.
- Implement the Cloudinary upload stream to securely store and serve heavy panoramic X-rays.

### Phase 5: Live Messaging & Immutable Audit Trails
- Setup Socket.io on the Express server to handle real-time patient-to-admin communications.
- Build the backend middleware that strictly logs all destructive actions (e.g., declining appointments) into the `audit_logs` table.
- Enforce the `403 Forbidden` response for any `DELETE /api/messages` attempts to guarantee an immutable trail.

### Phase 6: Gemini AI Engine
- Integrate the `@google/generative-ai` SDK.
- Build the specific system prompts for the Symptom Checker (patient-facing) and the Clinical Scribe (admin-facing).
- Secure the `/api/ai/*` routes behind the strict 12 req/min `aiLimiter`.
